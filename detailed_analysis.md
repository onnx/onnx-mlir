# test_linalg3.onnx.so 실행 결과 분석

## 실행 결과
- 입력 1: [1, 2, 3, 4, 5, 6, 7, 8] (shape: 2x4)
- 입력 2: [1, 1, 1, 1, 1, 1, 1, 1] (shape: 2x4)
- 출력: [3, 3, -6.47145e+30, 3.93405e+32, 11, 11, -1.51e+31, 9.17945e+32] (shape: 2x4)

## 분석

### 1. 부분적으로 맞는 값
- 출력[0] = 3 (정상 범위)
- 출력[1] = 3 (정상 범위)
- 출력[4] = 11 (정상 범위)
- 출력[5] = 11 (정상 범위)

### 2. 완전히 틀린 값
- 출력[2] = -6.47145e+30 (매우 큰 음수, 오버플로우)
- 출력[3] = 3.93405e+32 (매우 큰 양수, 오버플로우)
- 출력[6] = -1.51e+31 (매우 큰 음수, 오버플로우)
- 출력[7] = 9.17945e+32 (매우 큰 양수, 오버플로우)

## 가능한 원인

1. **시그니처 문제**: 입력/출력 시그니처가 "[ ]"로 비어있음
   - Linalg 파이프라인에서 시그니처 생성이 제대로 되지 않음
   - 이전에 확인한 대로 `addONNXToLinalgPasses`에서 시그니처를 `"[ ]@[ ]"`로 하드코딩

2. **메모리 초기화 문제**: 
   - 출력 버퍼가 제대로 초기화되지 않았을 수 있음
   - stride 계산이 잘못되어 잘못된 메모리 위치를 읽고 있을 수 있음

3. **입력 데이터 형식 문제**:
   - OMTensor 생성 시 stride가 잘못 설정되었을 수 있음
   - 메모리 레이아웃이 예상과 다를 수 있음

4. **MatMul 연산 문제**:
   - 행렬 곱셈의 인덱스 계산이 잘못되었을 수 있음
   - 일부 요소는 정상(3, 11)이지만 일부는 오버플로우
